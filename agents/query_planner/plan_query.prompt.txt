    ====================================================
    RELATIONSHIP RULES (CRITICAL — READ CAREFULLY)
    ====================================================

    Hasura has TWO types of relationships. They behave DIFFERENTLY:

    OBJECT relationships (many-to-one, e.g., menu_items.categories):
      - Returns a SINGLE related row.
      - Does NOT accept a 'where' argument at selection level.
      - To FILTER by an OBJECT relationship, use it in the PARENT table's where clause:
          menu_items(where: {{ categories: {{ name: {{ _ilike: "%Pizza%" }} }} }}) {{ ... }}
      - To SELECT fields from it, just nest the relationship name without where:
          menu_items {{ categories {{ name }} }}

    ARRAY relationships (one-to-many, e.g., categories.menu_items):
      - Returns MULTIPLE related rows.
      - DOES accept a 'where' argument at selection level:
          categories {{ menu_items(where: {{ is_vegetarian: {{ _eq: true }} }}) {{ name }} }}

    CRITICAL MISTAKES TO AVOID:
      WRONG — applying where on OBJECT relationship at selection level:
        menu_items {{ categories(where: {{ name: {{ _eq: "Pizza" }} }}) {{ id }} }}
      CORRECT — filtering via OBJECT relationship in parent where:
        menu_items(where: {{ categories: {{ name: {{ _ilike: "%Pizza%" }} }} }}) {{ categories {{ name }} }}

    IMPORTANT:
      - You MUST use the EXACT relationship name from the schema (e.g., 'categories', NOT 'category').
      - Relationship names match the Hasura-tracked names shown in the schema.

    ====================================================
    WHERE CLAUSE OPERATORS (Hasura)
    ====================================================

    Use only these operators:

    - _eq        → Exact match
    - _neq       → Not equal
    - _ilike     → Case-insensitive match (use % for wildcards)
    - _in        → Value in array
    - _gt, _gte  → Greater than
    - _lt, _lte  → Less than
    - _is_null   → Check if null
    - _and       → Logical AND
    - _or        → Logical OR
    - _not       → Logical NOT

    Rules:
    - Boolean fields use true/false (not strings)
    - Integer fields (Int) use whole numbers only (e.g., rating: 4, NOT 3.5)
    - Numeric/decimal fields (numeric) can use floats (e.g., price: 9.99)
    - Use _ilike for partial text searches
    - Use _eq for exact matches
